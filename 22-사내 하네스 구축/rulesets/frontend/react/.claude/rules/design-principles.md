# 설계 원칙

## 구조

- 동시에 실행되지 않는 코드는 분리된 컴포넌트로
- 함께 수정되는 파일은 같은 디렉토리에 (도메인별 분류)

## 가독성

- 복잡한 조건에 변수명 부여 (맥락 축소)
- 삼항연산자 중첩 금지, early return 사용
- 매직 넘버/스트링은 상수로 추출

## 예측 가능성

- 이름 겹침 방지 (같은 이름 = 같은 동작)
- 반환 타입 통일 (같은 종류 함수는 같은 형태)
- 함수 시그니처로 예측 불가능한 부작용은 명시화

## 결합도

- 과도한 공통화 금지 (동작이 다르면 중복 허용)
- 3번 반복되기 전까지 추상화하지 않음
