# 비개발자 Claude 프롬프트 레시피북 — 복붙해서 쓰는 업무 자동화 패턴 25선

> 최종 수정: 2026-04-03

## TL;DR

비개발자가 Claude Chat/Cowork에서 바로 쓸 수 있는 프롬프트 레시피 25개. 업무 유형별(문서/리서치/콘텐츠/데이터/회의)로 분류했고, `[대괄호]` 안만 바꿔서 복붙하면 된다. 좋은 프롬프트의 공식은 하나: **맥락 + 할 일 + 결과물 형식**.

---

## Part 1: 좋은 프롬프트의 공식

### 기본 구조

```
[맥락] 나는 [직무]이고, [상황]이다.
[할 일] [구체적인 작업]을 해줘.
[결과물] 결과는 [형식/길이/톤]으로 작성해줘.
[제약] [하지 말아야 할 것 / 포함해야 할 것]
```

### 나쁜 프롬프트 vs 좋은 프롬프트

| 나쁜 프롬프트 | 좋은 프롬프트 |
|---|---|
| "보고서 써줘" | "마케팅팀 주간 성과 보고서를 써줘. 항목: KPI 달성률, 주요 이슈, 다음 주 계획. 2페이지 이내, 임원 보고용 격식체." |
| "이메일 작성해줘" | "거래처 A사에 납기 지연 안내 이메일을 써줘. 사과 → 원인 → 대안 순서로, 정중하지만 간결하게, 3문단 이내." |
| "정리해줘" | "첨부된 회의록에서 액션 아이템만 추출해서 담당자 / 할 일 / 기한 형식의 표로 만들어줘." |

## Part 2: 레시피 25선

### 카테고리 1: 문서 작성 (5개)

**레시피 1 — 주간 업무 보고서**
```
나는 [직무]야. 아래 내용을 바탕으로 주간 업무 보고서를 작성해줘.

[이번 주 한 일]:
- 

[이슈/블로커]:
- 

[다음 주 계획]:
- 

형식: 항목별 글머리표, 임원 보고용 격식체, A4 1페이지 분량.
```

**레시피 2 — 채용공고 작성**
```
아래 내부 JD를 바탕으로 외부 채용공고를 작성해줘.

[내부 JD 내용]:
[여기에 붙여넣기]

회사 톤앤매너: [예: 스타트업 / 대기업 / 친근한 / 격식체]
플랫폼: [예: 링크드인 / 잡코리아 / 원티드]
포함 필수 항목: 주요 업무, 자격요건, 우대사항, 혜택.
```

**레시피 3 — 회의 전 브리핑 문서**
```
내일 [회의 주제] 회의가 있어. 아래 정보로 브리핑 문서를 만들어줘.

참석자: [이름/직책]
회의 목적: [의사결정 / 보고 / 워크샵]
배경 자료: [여기에 붙여넣기 또는 파일 첨부]

결과물: 핵심 쟁점 3개, 각 쟁점별 찬반 논거, 예상 질문 5개.
```

**레시피 4 — 제안서 초안**
```
[고객사명]에 제출할 [프로젝트명] 제안서 초안을 작성해줘.

고객사 현황: [문제/니즈]
우리의 솔루션: [핵심 제안]
차별점: [경쟁사 대비 강점]

구조: 문제 정의 → 솔루션 → 기대효과 → 진행 방식 → 비용/일정.
톤: 전문적이고 설득력 있게.
```

**레시피 5 — SOP(업무 매뉴얼) 작성**
```
아래 업무 과정을 표준 업무 매뉴얼(SOP)로 정리해줘.

업무명: [예: 신규 거래처 등록 프로세스]
현재 과정 (내가 하는 방식):
1. 
2. 
3. 

형식: 단계별 번호 리스트, 각 단계에 담당자/소요시간/주의사항 포함.
```

### 카테고리 2: 리서치 & 분석 (5개)

**레시피 6 — 경쟁사 분석**
```
[우리 회사/제품]의 경쟁사 분석 보고서를 작성해줘.

분석 대상: [경쟁사 A, B, C]
분석 항목: 주요 제품/서비스, 가격 정책, 강점, 약점, 최근 동향.

형식: 비교표 + 각 경쟁사 요약 1문단.
마지막에 우리가 취할 수 있는 전략적 시사점 3가지 추가.
```

**레시피 7 — 트렌드 리포트**
```
[산업/주제] 분야의 최신 트렌드를 조사해서 리포트를 작성해줘.

기간: 최근 [3개월 / 6개월 / 1년]
포함 항목: 핵심 트렌드 5개, 각 트렌드별 사례 2개, 우리 업무에 주는 시사점.
형식: 임원 보고용, 2페이지 이내.
```

**레시피 8 — 인터뷰/설문 인사이트 정리**
```
아래 [인터뷰 녹취록 / 설문 응답]에서 핵심 인사이트를 추출해줘.

[내용 붙여넣기]

결과물:
1. 공통으로 나온 의견 Top 5
2. 의외의 발견 (예상과 다른 점)
3. 즉시 실행 가능한 액션 아이템 3개
```

**레시피 9 — 데이터 해석**
```
아래 데이터를 해석해서 비전문가도 이해할 수 있는 인사이트로 변환해줘.

[데이터 붙여넣기 또는 파일 첨부]

목적: [예: 경영진 보고 / 팀 공유]
포인트: 숫자 나열 말고, "무엇을 의미하는지"를 중심으로 서술.
형식: 핵심 발견 3줄 요약 + 상세 설명.
```

**레시피 10 — 뉴스/아티클 요약**
```
아래 [기사/보고서/논문]을 요약해줘.

[링크 또는 내용 붙여넣기]

형식:
- 핵심 주장 1문장
- 주요 근거 3개 (글머리표)
- 우리 업무에 적용 가능한 점
길이: 200자 이내로 간결하게.
```

추출 항목: 날짜, 가맹점명, 금액, 카테고리(식비/교통/숙박/기타).

**레시피 17 — 파일 정리 자동화 (Cowork)**

조건:

## Part 3: Cowork 전용 — Skills 설정법

자주 쓰는 레시피는 **Skills**로 저장하면 매번 타이핑 안 해도 됨.

### Skills 저장 방법
1. Cowork 탭 → Settings → Skills
2. 이름 + 프롬프트 내용 저장
3. 다음부터 "주간보고서 만들어줘" 한 마디로 실행

### 추천 Skills 목록

| Skill 이름 | 트리거 | 내용 |
|---|---|---|
| 주간보고서 | "이번 주 보고서" | 레시피 1 고정화 |
| 회의록정리 | "회의록 정리" | 레시피 21 고정화 |
| 콘텐츠변환 | "SNS 변환" | 레시피 12 고정화 |
| 경비처리 | "경비 정리" | 레시피 16 고정화 |

---

## 핵심 인사이트

1. **"복붙 + [대괄호] 수정"이 전부**: 프롬프트 잘 쓰는 법 배울 필요 없다. 검증된 구조를 복붙하고 맥락만 채우면 된다.
2. **반복 업무는 Cowork + Skills로 고정**: 같은 프롬프트를 매주 치는 건 낭비. Skills에 저장하면 한 마디로 실행, 스케줄 걸면 알아서 실행.
3. **좋은 프롬프트 = 맥락 + 할 일 + 결과물 형식**: 이 세 가지만 있으면 어떤 업무든 80점 이상 나온다.
